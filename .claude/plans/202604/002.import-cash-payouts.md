# 002 — Импорт «Выплаты наличные» (xls)

## Проблема

CLI должен читать xls-реестр «Выплаты наличные» (Дата|ФИО|Сумма|Коммент), создавать в БД один `TransactionGroup` и по одной `Transaction` (расход) на строку. Счёт — из config по имени. Категории и дедуп — не трогаем. Параллельно уходим от AOT и переключаемся на EF Core, работаем с копией рабочей БД в `.temp/`.

## Шаги

1. Скопировать БД в `.temp/Ledmaster.cash`, дописать `.gitignore`, переключить `config.json`.
2. В csproj: снять AOT-флаги, добавить пакеты (через `dotnet add package`). Зарегистрировать `CodePagesEncodingProvider` в `Program.cs` (для .xls).
3. `dotnet ef dbcontext scaffold "Data Source=.temp/Ledmaster.cash" Microsoft.EntityFrameworkCore.Sqlite -o src/Data/Entities --context-dir src/Data --context AppDbContext --no-onconfiguring`.
4. Ревизия scaffold: добавить value converters — `Guid` (если TEXT), `Changed` UNIXTIME → `DateTimeOffset`, `MONEY` → `decimal`, `Deleted` → `bool`. `OnModelCreating` держим в `AppDbContext`, entities — partial при необходимости.
5. `AppConfig.ImportAccountName` + `AppConfigJsonContext` обновить. `CliOptions.ImportPath` + парсер `--import`.
6. `IImporter` + `ImportRecord` + `CashPayoutsImporter`: открыть .xls, найти первый лист, пропустить шапку до строки с «Дата/ФИО/Сумма», читать до первой пустой. Excel-serial → `DateTime.FromOADate`.
7. `TransactionWriter.Write(IEnumerable<ImportRecord>, string accountName)`: lookup `Account` (ошибка если нет), создать `TransactionGroup` (`HolderDateTime`=min(Date), `Position`=next), по записи — `Transaction` (`ExpenseAccount`=Account.Id, `ExpenseAmount`=record.Amount, `Comment`=`{Person} — {Comment}` или раздельно, `BudgetDate`=Date, `Executed`=Date, `Guid`=new, `Changed`=now). Всё в одной `SaveChangesAsync` внутри транзакции.
8. `Program.cs`: если `--import` задан — открыть DbContext из `DbPath`, вызвать importer, writer; иначе текущее поведение (печать DbPath).

## Риски

- **Scaffold переписывает файлы.** Повторный запуск затирает правки конвертеров — делаем один раз, дальше руками. Если понадобится пересканировать — через отдельную ветку/worktree.
- **Excel-serial дата:** база 1899-12-30, `DateTime.FromOADate` — учесть.
- **cp1251 в .xls:** без `CodePagesEncodingProvider.Instance` будут крашиться SST-записи.
- **Сопоставление колонок хрупкое:** шапка в «Выплаты наличные» начинается на строке 3, данные с 5-й. Жёстко зашивать индексы — плохо; матчим по заголовкам («Дата», «Сумма» и т.п.) с нормализацией.
- **Модель `Transaction`:** часть полей (`ExtraComment1..4`, `Quantity`, `BudgetPeriodEnd`, `Locked`, `IncomeBalance`/`ExpenseBalance`) — не nullable в скаффолде но по факту бывают NULL. Сверить nullable после scaffold.
- **FK `TransactionGroups.Id`:** `Transaction.Group` — NOT NULL; сначала save группы → получить Id → потом транзакции. Или `TransactionGroups` + navigation collection в одном `SaveChanges`.
- **`config.json` закоммичен.** Сейчас в `.gitignore` есть `AbilityCashCli/config.json` — OK, дефолт пересоздаётся `ConfigStore`. Новое поле `ImportAccountName` добавляем в код, локальный файл юзер правит сам.
