# 014 — TransactionGroup 1:1 с Transaction

## Проблема

Писатели создают одну `TransactionGroup` на весь файл и вешают туда все транзакции; AbilityCash ожидает 1:1 и падает при открытии БД (группа с 410 транзакциями).

## Файлы

- `AbilityCashCli/src/Import/BankStatements/BankStatementWriter.cs` — группа per row.
- `AbilityCashCli/src/Import/CashPayoutsWriter.cs` — группа per row.
- `AbilityCashCli/src/Import/Timesheets/TimesheetWriter.cs` — группа per row.
- `AbilityCashCli/src/Import/SalaryRegisters/SalaryRegisterWriter.cs` — группа per row.
- `AbilityCashCli/src/Import/Vacations/VacationsWriter.cs` — группа per row.

## Шаги

1. В каждом писателе: вычислить стартовый `Position` по `HolderDateTime` один раз до цикла, вести `Dictionary<int,int>` next-Position per `HolderDateTime`.
2. Внутри цикла — создать новую `TransactionGroup` с `HolderDateTime = BudgetDate` строки и `Position` из словаря (инкремент), сразу назначить `transaction.GroupNavigation = group`.
3. `Guid`, `Changed`, `Deleted=0`, `Recurrence=null` — как сейчас.
4. Убрать единую группу на файл и блок `_db.TransactionGroups.Add(group)` в конце — EF добавит группы через навигацию.
