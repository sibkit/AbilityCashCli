# 019 — Alfa-Bank реестры начислений

## Проблема

Новый тип файла — `Реестр начислений_*.xls`: зарплатный реестр Альфа-Банка для ООО Ледмастер. Источник — `[ЗП] Ледмастер (Альфа)`, dest — `[З] ФИО`. Имя предприятия лежит только в содержимом (row 4: `Общество с ограниченной ответственностью "Ледмастер"`), в имени файла его нет — текущий `SalaryRegisterWriter.ResolveEnterprise` (substring в source) не сработает.

## Файлы

- `src/Import/ImportRecord.cs` — добавить `EnterpriseHint: string?` (nullable).
- `src/Import/SalaryRegisters/SalaryRegisterWriter.cs` — в `ResolveEnterprise`: если у `records[0]` задан `EnterpriseHint`, матчим `enterprise.Name.Contains(hint, OrdinalIgnoreCase)`; иначе текущее `source.Contains(enterprise.Name, ...)`.
- `src/Import/SalaryRegisters/AlfaBankSalaryRegisterImporter.cs` — `.xls`, ExcelDataReader. Извлекает: enterprise hint (row 4 — текст в двойных кавычках, fallback: вся непустая ячейка), дату (row 11 col «от»), назначение (row 12 col «Назначение платежа»), № реестра (row 2 col «Реестр №»). Шапка — по маркерам `№№`, `ФИО работника`, `№ текущего счета`, `Сумма перевода`, `Сумма удержанных средств`. Данные — пока № заполнен (до строки `Итого:`). Comment = `{purpose} | реестр {regNumber}`.
- `src/Import/SalaryRegisters/AlfaBankSalaryRegisterHandler.cs` — `MatchesByName`: имя начинается с `Реестр начислений` и расширение `.xls`. Writer — общий `SalaryRegisterWriter`.
- `src/Program.cs` — DI: новый importer+handler, регистрация в `ImportRouter`.

## Шаги

1. `ImportRecord` + опциональный `EnterpriseHint`.
2. `SalaryRegisterWriter` — учёт `EnterpriseHint`, двусторонний выбор стратегии резолва.
3. `AlfaBankSalaryRegisterImporter` — парсинг шапки/данных.
4. `AlfaBankSalaryRegisterHandler` — match по имени, wiring с writer.
5. `Program.cs` — DI, регистрация в router.

## Риски

- **`EnterpriseHint` в общем record** — используется только salary-писателями; остальные (timesheet, vacation, cash payouts) игнорируют. Минимально инвазивно.
- **Extract quoted name** — если row 4 сверстан иначе (без кавычек), fallback на всю строку ячейки; резолв ent.Name.Contains продолжит работать при содержательном совпадении.
- **Дубль со строками выписки Альфа** — ожидаемо, пользователь правит вручную (по CLAUDE.md). Не дедуплицируем.
- **Статус «ИСПОЛНЕНО»** в файле присутствует блоком, а не per-row — проверять глобально не обязательно (TБанк проверяет per-row из-за иной семантики). Пропускаем.
