# 006 — T-Банк зарплатные реестры + концепт Enterprise

## Проблема

Новый тип файла — выгрузка зарплатного проекта T-Банка (`Vyplaty po 3 reestram *_<enterprise>.xlsx`): каждая строка = перевод сотруднику. Источник — `[ЗП]`-счёт предприятия; dest — `[З] ФИО`. Предприятие (ИП Желтухин, ООО Ледмастер) — новый программный концепт, AbilityCash про него не знает. Формат колонок зависит от банка — importer per-bank, writer общий.

## Риски

- **Rule открывает файл** — I/O на каждый `Matches`. Файлов мало, терпимо.
- **Enterprise-матч по substring** — если имена пересекаются (сейчас не пересекаются: «ИП Желтухин» ∩ «Ледмастер» = ∅), писатель ругнётся на множественное совпадение.
- **Статус != ИСПОЛНЕН → throw** — файл не архивируется, ручной разбор.
- **ЗП-счёт Ледмастера (Альфа)** в конфиге — будет использован будущим Alfa-importer-ом; T-Банк-rule этот файл не сматчит.
