# 004 — Импорт отпусков

## Проблема

Нужен importer для `Отпуска*.xlsx`: по каждому сотруднику резолвим счёт `{SalaryPrefix}{Имя}`, считаем среднемесячную ЗП за расчётное окно 12 мес. перед месяцем отпуска (полные месяцы) по категориям-начислениям (`Расход` на счёт, `IncomeAccount IS NULL`), делим на `AverageDaysPerMonth` (29.3), умножаем на дни отпуска — получаем сумму отпускных. Пишем одну `Transaction` с `TransactionCategory`=путь категории отпускных. Писатель требует доступ к БД (в отличие от cash-payouts). Рефакторим пайплайн: importer+writer привязываются к rule'у.

## Шаги

1. CLAUDE.md: короткий абзац «доменные константы — в config, не литералом».
2. `AppConfig.Vacation` + `CreateDefault` (пути-плейсхолдеры в дефолте). Путь категории — строка, separator из `CategoryPathSeparator`.
3. Рефакторинг пайплайна: `IImportWriter`, `IImportRule.Writer`, `IImportRouter.Resolve→IImportRule?`, `ImportRouter` — first-match-wins возвращает rule. `TransactionWriter` → `CashPayoutsWriter` (implements `IImportWriter`, accountName в ctor). `CashPayoutsRule` получает importer+writer.
4. `CategoryPathResolver(AppDbContext, separator)`: разбивает путь по separator, идёт по `Categories` через `Parent` (корень — `Parent IS NULL AND Name=part0`, дальше по `Parent=prev.Id`), `Deleted=0`, точное совпадение `Name`. Нет совпадения → `InvalidOperationException`. Кэш на время writer'а (resolve один раз в `WriteAsync`).
5. `VacationsImporter.Read`:
   - Шапка row 1: `ФИО | Даты | Кол-во дней`. Жёстко маппим по именам с trim+ci.
   - Строка: `Person = NormalizePersonName(ФИО)` (trim + сворачиваем пробел после точки: `. ` → `.`, чтобы «Смирнова Т. Г.» → «Смирнова Т.Г.» — совпадало с форматом `Accounts.Name`). `dates = "dd.mm.yyyy-dd.mm.yyyy"` → `startDate`, `endDate`, `days = Кол-во дней`.
   - `ImportRecord(Date=startDate, Amount=days, Person=Person, Comment=$"Отпуск с {startDate:dd.MM.yyyy} {days} дней")`. `Amount` — транзитно (writer подставит реальную сумму).
6. `VacationsWriter.WriteAsync(source, records, ct)`:
   - `vacationCategoryId = resolver.Resolve(cfg.VacationCategoryPath)` — throw если нет.
   - `salaryCategoryIds = cfg.SalaryCategoryPaths.Select(resolver.Resolve).ToHashSet()` — throw если нет.
   - Один `TransactionGroup`: `HolderDateTime = nowUnix`, `Position = next` (max Position для `HolderDateTime=nowUnix` + 1), `Guid`, `Changed`, `Deleted=0`.
   - Для каждой record:
     - `accountName = cfg.SalaryAccountPrefix + r.Person`
     - Account lookup; нет → skip+log, continue.
     - `vacStart = r.Date`, `days = (int)r.Amount`.
     - accrualsQuery: `Transactions` where `ExpenseAccount=acc.Id AND IncomeAccount IS NULL AND Deleted=0 AND TransactionCategories.Any(tc => salaryCategoryIds.Contains(tc.Category) && tc.Deleted=0)`.
     - `workStart = min(BudgetDate)` по accrualsQuery (unix→DateTime); если null → skip+log «нет начислений», continue. (`Executed` — это флаг 0/1, дата проводки в `BudgetDate`.)
     - Расчётное окно: `windowEnd = последний день месяца перед vacStart`, `windowStart = windowEnd.AddDays(1).AddMonths(-12)`. `workStartMonth = startOfMonth(workStart); if workStart.Day > 1 → workStartMonth.AddMonths(1)`. `effectiveWindowStart = max(windowStart, workStartMonth)`.
     - `monthsCount = (windowEnd.Year - effectiveWindowStart.Year)*12 + (windowEnd.Month - effectiveWindowStart.Month) + 1`. Если ≤0 → skip+log.
     - `sum = accruals.Where(BudgetDate в [effectiveWindowStart, windowEnd]).Sum(|ExpenseAmount|)`. Если `sum == 0` → skip+log.
     - `avgMonth = sum / monthsCount` (decimal), `amount = avgMonth / cfg.AverageDaysPerMonth * days`.
     - `dateUnix = startOfDayUnix(vacStart.AddDays(-cfg.PreDaysOffset))`.
     - `group.Transactions.Add(new Transaction { ExpenseAccount=acc.Id, ExpenseAmount=-ToStoredAmount(amount), Executed=1, BudgetDate=dateUnix, BudgetPeriodEnd=dateUnix+DaySeconds, Quantity=QuantityOne, Comment=r.Comment, ExtraComment2=source, TransactionCategories={ new { Category=vacationCategoryId, ... } } ... })`. `Executed=1` — флаг «проведено», как у cash-payouts и существующих начислений.
   - `db.TransactionGroups.Add(group)`; `BeginTransaction` → `SaveChanges` → `Commit`. Возвращает `saved`.
   - Если после фильтрации ни одной транзакции не создано → группу не сохраняем, возвращаем 0.
7. `VacationsRule(VacationsImporter, VacationsWriter)` + regex.
8. `BulkImportRunner`: заменить `_writer` на `_router → rule.Writer`; по ходу собирать counters+итоговые списки; в конце — `summary(imported, skipped, errored + перечислить по имени файла)`. Exception на файле — в список errored. «Skip (нет importer'а)» — в skipped.
9. `Program.cs`: `cashRule = new CashPayoutsRule(new CashPayoutsImporter(), new CashPayoutsWriter(db, config.ImportAccountName))`. `vacRule = new VacationsRule(new VacationsImporter(), new VacationsWriter(db, config.Vacation, new CategoryPathResolver(db, config.Vacation.CategoryPathSeparator)))`. Router на этих двух.

## Риски

- **Category path separator `::`** — если у категории в имени встретится `::`, сломается. Маловероятно; документируем через дефолт в config.
- **`BudgetDate` NOT NULL** — ставим = `dateUnix` (день выдачи отпускных).
- **Откат при fail-fast категории.** `BeginTransaction` → resolve категорий — throw до `Add`. Транзакции не было — откатывать нечего. Если throw после `SaveChanges`, `Commit` не вызовется → rollback на Dispose. OK.
- **Одна DB-транзакция на файл.** Вся file-level консистентность; отдельные сотрудники skip'аются без падения файла.
- **Float в формуле.** Всё decimal; `amount * 10000` округляется через `AbilityCashValues.ToStoredAmount` (MidpointRounding.AwayFromZero).
- **Lookup начислений по `TransactionCategories`.** EF-запрос со `Any` на коллекции — SQLite переводит нормально; проверяем перформ на 24-мес окне: индекс по `Transactions.ExpenseAccount` и `TransactionCategories.Transaction` желательно, но не в скопе плана.
