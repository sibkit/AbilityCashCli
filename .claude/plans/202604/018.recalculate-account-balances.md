# 018 — Пересчёт IncomeBalance/ExpenseBalance

## Проблема

AbilityCash не пересчитывает `Transactions.IncomeBalance`/`ExpenseBalance` сам — это денормализованный running balance после каждой операции. Наши импорты оставляют NULL, остатки в UI врут.

## Файлы

- `AbilityCashCli/src/Import/AccountBalanceRecalculator.cs` — новый класс. Пересчитывает балансы для всех не-удалённых транзакций.
- `AbilityCashCli/src/Import/BulkImportRunner.cs` — инъекция `AccountBalanceRecalculator`, вызов перед `trx.CommitAsync`.
- `AbilityCashCli/src/Program.cs` — подстановка зависимости.
- `CLAUDE.md` — секция про домен: `IncomeBalance`/`ExpenseBalance` = остаток счёта после транзакции, пересчитывается целиком перед коммитом импорта.

## Шаги

1. `AccountBalanceRecalculator.RecalculateAsync`:
   - Загрузить все `Accounts` (включая удалённые), словарь `Id → StartingBalance`.
   - Загрузить все `Transactions` (не удалённые) с `Include(GroupNavigation)`, упорядочить: `Group.HolderDateTime`, `Group.Position`, `Transaction.Position`, `Id` — именно этот порядок использует AbilityCash при пересчёте на сохранении. `BudgetDate` как первичный ключ не годится: в легаси-группах `HolderDateTime` — реальное время операции, а `BudgetDate` — всегда 00:00 UTC, и внутри суток порядок расходится.
   - Пройти один раз: на каждой транзакции обновить баланс счёта (`IncomeAmount` и/или `ExpenseAmount`) и записать новое значение в `IncomeBalance`/`ExpenseBalance`.
   - `SaveChangesAsync` — EF сам пропустит строки без изменений.
2. В `BulkImportRunner.RunAsync` вызвать recalc до `CommitAsync`, только если `erroredFiles.Count == 0` и что-то импортировали.
3. Обновить `CLAUDE.md`.

## Риски

- Если старые `IncomeBalance`/`ExpenseBalance` считались по другому порядку (другой tie-breaker внутри суток) — пересчёт может сдвинуть значения. Для пользователя это скорее плюс (единая логика), но надо знать.
- Если AbilityCash открыт на старой БД — там могут быть отличия. Свежий экспорт закрывает вопрос.

## Доработка: сортировка по HolderDateTime

Изначально был ошибочно выбран `BudgetDate` как первичный ключ сортировки. После наблюдения: открытие/сохранение транзакции в AbilityCash переписывает balance-колонки всего счёта по порядку `HolderDateTime, Group.Position, ...`. На легаси-данных `HolderDateTime != BudgetDate` у ~30% строк (реальное время операции vs 00:00 UTC), и running balance расходится внутри суток. Исправлено: порядок в `AccountBalanceRecalculator` — `Group.HolderDateTime, Group.Position, Transaction.Position, Id`. На свежих импортах эффекта нет (allocator ставит `HolderDateTime = BudgetDate`).
