## Описание

CLI-инструмент для автоматической обработки БД: импорт из различных форматов файлов, трансформация и сохранение данных.

используем систему планов

## Доменная модель AbilityCash

Три типа операций (одна строка `Transactions`):
- **Расход** — `ExpenseAccount` задан, `IncomeAccount` NULL. Статья (`TransactionCategory`) обязательна.
- **Доход** — `IncomeAccount` задан, `ExpenseAccount` NULL. Статья обязательна.
- **Перевод** — оба счёта заданы. Статья опциональна.

Понятие «бюджет» не используем. Поля `BudgetDate` / `BudgetPeriodEnd` в схеме — исторические; трактуем как дату операции и конец её периода. `BudgetDate` — всегда 00:00 UTC (`StartOfDayUnix`); порядок внутри суток — через `TransactionGroup.Position`.

`TransactionGroup` — 1:1 с `Transaction`. Каждая транзакция — своя группа; при нескольких транзакциях в одной группе AbilityCash падает при открытии БД. `HolderDateTime` = `BudgetDate` транзакции, `Position` = порядок среди групп с тем же `HolderDateTime`. `Recurrence` — всегда `"{}"` (`AbilityCashValues.RecurrenceEmpty`); NULL валит AbilityCash при открытии. Общий аллокатор — `TransactionGroupAllocator`.

**Enterprise** — программный концепт предприятия (`Name` + `PayrollAccount`). В AbilityCash отсутствует; используется на уровне CLI для роутинга источника зарплаты (например, по подстроке имени файла).

## Соглашения импорта

- **Префиксы счетов:** `[ЗП]` — зарплатный счёт предприятия (источник выплат), `[З]` — личный счёт сотрудника (формат `[З] ФИО`).
- **Суффикс `[..xxxx]`** в `Account.Name` — последние 4 цифры номера счёта в банке. Используется для сопоставления банковской выписки (где в имени файла есть полный 20-значный счёт) с записью в AbilityCash. Пример: `[ЗП] ИП Желтухин (Т-Банк) [..5852]`.
- **ExtraComment1** = Документ (у выписок: `№{№} от {дата}, ИНН {ИНН}`; у остальных писателей — `""`). **ExtraComment2** = Источник: `{filename} | {ImporterClassName} v{AppVersion}` (через `AbilityCashValues.BuildSourceComment`). Версия — `AppInfo.Version`. **ExtraComment3** и **ExtraComment4** — всегда `""` (в UI скрыты).

## Зарплатный поток

Зарплата сотрудника попадает в AbilityCash через несколько независимых источников, сходящихся на личный счёт `[З] ФИО`:

- **Табели** (`TimesheetHandler`, `.xls`) — расход по статье «оклад» с `[ЗП]` предприятия на `[З] ФИО`, пропорционально отработанным часам.
- **Реестры выплат ТБанка** (`TBankSalaryRegisterHandler`, `.xlsx`) — расход с `[ЗП]` предприятия, перевод на `[З] ФИО`.
- **Кассовые выплаты** (`CashPayoutsHandler`, `.xls`) — перевод из кассы или `[ЗП]` на `[З] ФИО`.
- **Отпуска** (`VacationsHandler`, `.xlsx`) — расход по статье «отпускные», перевод на `[З] ФИО`.

Банковские выписки (`AlfaBankHandler`, `TBankStatementHandler`) — отдельный поток: импортируют операции по корпоративному счёту. Зарплатные строки в выписке создают визуальный дубль со строками из реестра — пользователь правит вручную.

## Роли файлов и привязки

Источник распознаёт `IImportHandler` в два этапа через `ImportRouter`:

1. **По имени** (`MatchesByName`) — высокоприоритетные кандидаты. Проходим по ним по порядку; первый non-null `TryImportAsync` побеждает.
2. **По шапке** — если этап 1 не дал результата, проходим по остальным handler'ам; `TryImportAsync` сам проверяет расширение + маркер-колонки и возвращает `null`, если файл не его.

Привязки:

- Касса: `.xls` c именем `Выплаты наличные*` → `CashPayoutsHandler`.
- Отпуска: `.xlsx` c именем `Отпуска*` → `VacationsHandler`.
- Табель: `.xls[x]` c именем `Табель *` → `TimesheetHandler`.
- Реестры ТБанка: `.xlsx` с маркерами реестра → `TBankSalaryRegisterHandler`.
- Выписка Альфа-Банка: `.csv` TSV CP1251 с англ. маркер-колонками → `AlfaBankHandler`.
- Выписка Т-Банка: `.csv` `;`-separated UTF-8 с рус. маркер-колонками. Имя с 20-значным счётом даёт приоритет по имени, но финальное решение — по шапке → `TBankStatementHandler`.

Счёт выписки резолвится по суффиксу `[..xxxx]` в `Account.Name` — последние 4 цифры полного счёта из имени файла. `0` или `>1` совпадений — ошибка импорта.

## Обнови файлы

По команде «обнови файлы»:
1. Вернуть файлы из `AbilityCashCli/bin/**/imported/` в `.temp/_extracted/` (move).
2. Скопировать `D:\Ledmaster\ledmaster-git\Ledmaster.cash` → `.temp/Ledmaster.cash` (overwrite).

## Сборка и тесты

Claude **не запускает** `dotnet build/test/run` — пишет «запусти `dotnet build` / `dotnet test`, кинь итог» и ждёт. Исключение — явное разрешение в текущей переписке; тогда подавлять output: `dotnet build --verbosity quiet --nologo 2>&1 | tail -30`, аналогично для `test`.

## Конвенции

- **Src layout:** исходники `AbilityCashCli/src/`, тесты `AbilityCashCli.Tests/src/`.
- **SOLID:** зависимости через интерфейсы и DI, не прямыми ссылками на реализации.
- **Комментарии:** по умолчанию не пишем. Если нужен — на русском.
- **ProjectReference:** не добавлять новые `<ProjectReference>` без явного разрешения. Сначала предложи, объясни зачем, дождись подтверждения.

## Конфигурация и константы

Литералы-конфигурация (пути, таймауты, форматы) — в Options-record, не в телах методов. Connection strings и креды — только из `IConfiguration`/env, никогда литералом.

Доменные константы (имена/пути категорий, префиксы счетов, коэффициенты расчёта) — через `AppConfig`, не литералом в коде.
