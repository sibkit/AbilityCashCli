# 007 — Атомарный прогон архива, ФИО→инициалы, красные ошибки

## Проблема

Три правки: (1) ФИО в полной форме (`Чурбанова Анастасия Юрьевна`) не мапится на счёт `[З] Чурбанова А.Ю.`; (2) при ошибке в одном файле архива уже сохранённые файлы остаются в БД и архивируются; (3) ошибки в консоли не выделяются цветом.

## Файлы

- `src/Import/PersonNameNormalizer.cs` — если токенов ≥ 3: фамилия = всё до последних двух (join пробелом), последние два → инициалы (`А.Ю.`, без пробелов между). Токены, уже содержащие `.`, не сокращаем. При <3 токенах — текущее поведение (только DotSpace).
- `src/Import/BulkImportRunner.cs` — принимает `AppDbContext` напрямую. Открывает одну `BeginTransactionAsync` на весь прогон. Копит успешные файлы в локальный список (без архивации). По ошибке — `errored.Add` + красная строка через приватный `WriteError`, продолжает проход (чтобы вывести все ошибки). По окончании: `errored.Count > 0` → транзакция диспозится без commit (rollback), ничего не архивируется; иначе → `CommitAsync` и потом `_archiver.Archive` по каждому успешному файлу. Итоговый Summary печатается всегда.
- `src/Import/CashPayoutsWriter.cs` — убрать `BeginTransactionAsync`/`CommitAsync`, оставить `SaveChangesAsync` (нужен, чтобы следующий файл видел `position`/`HolderDateTime`).
- `src/Import/Vacations/VacationsWriter.cs` — то же.
- `src/Import/SalaryRegisters/SalaryRegisterWriter.cs` — то же.
- `src/Program.cs` — передать `db` в `BulkImportRunner`; catch-блоки `ArgumentException`/`Exception` — печать через helper (красный, в `Console.Error`), можно инлайн.

## Шаги

1. `PersonNameNormalizer` — новая логика + текущий DotSpace как пост-шаг на результате.
2. Writer'ы — снять внутренние транзакции.
3. `BulkImportRunner` — `AppDbContext`-зависимость, outer transaction, отложенная архивация, `WriteError` с `ConsoleColor.Red` + `Console.Error`.
4. `Program.cs` — DI, красные catch-сообщения.

## Риски

- **Длинная транзакция SQLite** — файлов единицы, терпимо.
- **Двойная фамилия** (`Петров-Водкин Иван Сергеевич`) — 3 токена, корректно: `Петров-Водкин И.С.`.
- **Составная фамилия из двух слов** (`Иванова Петрова Мария Сергеевна`, 4 токена) — «фамилия» = `Иванова Петрова`, инициалы = `М.С.`. Если в данных такое не встречается — вопрос теоретический.
- **EF Core запрет на вложенный `BeginTransactionAsync`** — поэтому снимаем их из writer'ов полностью.
- **Skipped-файлы** (нет importer'а / пусто) при ошибке в другом файле тоже не архивируются — консистентно «всё или ничего».
