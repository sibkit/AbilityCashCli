# 016 — Единообразные ExtraComments в писателях

## Проблема

Писатели пишут `ExtraComment3`/`ExtraComment4` по-разному: `BankStatementWriter` кладёт в `ExtraComment3` поле `ExtraDoc` (из TBank), остальные оставляют пустым. В настройках `Properties` видны только `ExtraComment1` (Документ) и `ExtraComment2` (Источник). Невидимые поля создают шум при сравнении импортированных записей с записями, созданными через UI.

## Файлы

- `AbilityCashCli/src/Import/AbilityCashValues.cs` — `BuildSourceComment`: защитно брать `Path.GetFileName(filename)`, чтобы в `ExtraComment2` гарантированно попадало только имя файла.
- `AbilityCashCli/src/Import/BankStatements/BankStatementRow.cs` — удалить поле `ExtraDoc`.
- `AbilityCashCli/src/Import/BankStatements/BankStatementWriter.cs` — `ExtraComment3 = ""`, убрать обращение к `r.ExtraDoc`.
- `AbilityCashCli/src/Import/BankStatements/Alfa/AlfaBankHandler.cs` — убрать `ExtraDoc: ""` из конструктора `BankStatementRow`.
- `AbilityCashCli/src/Import/BankStatements/TBank/TBankStatementHandler.cs` — убрать `ExtraDoc:` из конструктора и вычисление `description`, оно больше не используется.
- `CLAUDE.md` — зафиксировать инвариант `ExtraComment1`=Документ, `ExtraComment2`=Источник, 3/4 — всегда `""`.

## Шаги

1. Удалить поле `ExtraDoc` из `BankStatementRow` и зачистить обоих handler'ов (Alfa, TBank) — выкидывает неиспользуемую инфу; в комментарии и так дублируется.
2. `BankStatementWriter`: `ExtraComment3 = ""`, `ExtraComment4 = ""` (как у всех остальных).
3. `BuildSourceComment`: обернуть первый аргумент в `Path.GetFileName` — защита от случайного пути.
4. Обновить CLAUDE.md.

## Риски

`r.ExtraDoc` в TBank иногда содержит строку, не совпадающую с `Comment`. Пользователь подтвердил: информация дублируется в `Comment`, потеря допустима.
