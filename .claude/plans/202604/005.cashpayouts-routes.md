# 005 — Выплаты наличные: роуты и перевод вместо расхода

## Проблема

Глобальный `ImportAccountName` не подходит: у каждого типа файла свой набор счетов и правил. Для cash-payouts нужен per-file-type конфиг с роутами по коммету (source-счёт выбирается по `Comment`), dest — ЗП-счёт сотрудника (`[З] ФИО`), транзакция — перевод (expense+income), время операции — `DefaultTime` из конфига.

## Файлы

- `AbilityCashCli/src/Configuration/AppConfig.cs` — удалить `ImportAccountName`, добавить `CashPayouts: CashPayoutsConfig`.
- `AbilityCashCli/src/Configuration/CashPayoutsConfig.cs` — `SalaryAccountPrefix`, `DefaultTime: string "HH:mm"`, `Routes: IReadOnlyList<CashPayoutRoute>`; `CashPayoutRoute(Comment, SourceAccount)`.
- `AbilityCashCli/src/Import/PersonNameNormalizer.cs` — общий `Normalize(string)`: trim + regex `\.\s+` → `.`.
- `AbilityCashCli/src/Import/Vacations/VacationsImporter.cs` — переключить на общий `PersonNameNormalizer`.
- `AbilityCashCli/src/Import/CashPayoutsImporter.cs` — применить `PersonNameNormalizer` к ФИО. Date отдавать as-read (без искусственного времени).
- `AbilityCashCli/src/Import/CashPayoutsWriter.cs` — полный rewrite: роут по `Comment.Trim()` + `OrdinalIgnoreCase`, exact match; throw если нет. Dest = `SalaryAccountPrefix + Person`. Перевод-транзакция (`ExpenseAccount=src`, `IncomeAccount=dst`, `ExpenseAmount=-stored`, `IncomeAmount=+stored`, `Executed=1`, `BudgetDate = ToUnix(Date + (TimeOfDay==0 ? DefaultTime : TimeOfDay))`, `BudgetPeriodEnd = BudgetDate + DaySeconds`, `Comment = Person`, `ExtraComment2 = source`).
- `AbilityCashCli/src/Import/AbilityCashValues.cs` — `int ToUnix(DateTime)` (полный timestamp, не round-to-day).
- `AbilityCashCli/src/Program.cs` — убрать check `ImportAccountName`, `CashPayoutsWriter(db, config.CashPayouts)`.
- `AbilityCashCli/config.json` — убрать `ImportAccountName`, добавить `CashPayouts`.

## Шаги

1. `AppConfig`: убрать `ImportAccountName`, добавить `CashPayoutsConfig` + `CashPayoutRoute`. Обновить `CreateDefault`.
2. `PersonNameNormalizer` (статический). `VacationsImporter` — заменить локальный normalize на общий.
3. `AbilityCashValues.ToUnix(DateTime)`.
4. `CashPayoutsImporter` — normalize person, Date as-read.
5. `CashPayoutsWriter` rewrite. Роут (`Trim + OrdinalIgnoreCase` exact) или throw. Dest lookup (`[З] Person`), Source lookup (route.SourceAccount). `BudgetDate` с временем (DefaultTime если TimeOfDay==0). Единая group на файл, `HolderDateTime = min(BudgetDate)`, `Position = max+1`. File-level DB-транзакция.
6. `Program.cs` — удалить `ImportAccountName` check/usage, прокинуть `config.CashPayouts` в writer.
7. `config.json` — обновить под новую схему.

## Риски

- **Неизвестный коммент → throw** — DB-транзакция ещё не открыта (throw до SaveChanges), откатывать нечего; файл остаётся в исходной папке (не архивируется), пользователь добавляет route и перезапускает.
- **DefaultTime = 13:00** — если в будущем в файле появится явное время, writer его уважает (берёт `TimeOfDay`, если != 0).
- **Общий PersonNameNormalizer** — если в будущем xlsx-ы разных типов потребуют разный стиль, выделим per-type.
