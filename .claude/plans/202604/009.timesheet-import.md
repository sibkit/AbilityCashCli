# 009 — Импорт табелей

## Проблема
Импортировать табели (`Табель {месяц} {год}.xls`) как расход с личного счёта `[З] ФИО` по статье «Оклад». Сумма = оклад из конфига × отработанные часы / норма часов; норма берётся из самого файла. Сотрудник к предприятию не привязан.

## Файлы
- `AbilityCashCli/src/Configuration/TimesheetConfig.cs` — новый record: `SalaryAccountPrefix`, `SalaryCategoryPath`, `DefaultTime`.
- `AbilityCashCli/src/Configuration/SalaryConfig.cs` — новый record: `Person`, `Amount`.
- `AbilityCashCli/src/Configuration/AppConfig.cs` — добавить `Timesheet` и `Salaries`; обновить `CreateDefault()`.
- `AbilityCashCli/src/Configuration/AppConfigJsonContext.cs` — обновления не требуются (generic resolver уже знает вложенные типы через `AppConfig`).
- `AbilityCashCli/src/Import/Timesheets/TimesheetImporter.cs` — парсер xls через `ExcelDataReader`: дата периода из ячейки B2, шапка по строкам `Фамилия,инициалы…/Отработано…/норма`, строки данных; возвращает `ImportRecord(Date=lastDayOfMonth+DefaultTime, Amount=hours/norm, Person, Comment)`; фатал при норме=0, часах<0 или отсутствии заголовка.
- `AbilityCashCli/src/Import/Timesheets/TimesheetWriter.cs` — по ФИО ищет оклад в `Salaries` (фатал, если нет) и счёт `[З] ФИО` в БД (фатал, если нет); создаёт одну `TransactionGroup`, по строке — расход с `ExpenseAccount=account`, `IncomeAccount=null`, округление до копеек; категория по `SalaryCategoryPath`.
- `AbilityCashCli/src/Import/Timesheets/TimesheetRule.cs` — `Matches` по regex `^Табель\s.*\.xlsx?$` (IgnoreCase).
- `AbilityCashCli/src/Program.cs` — собрать и зарегистрировать `TimesheetRule` в `ImportRouter`.
- `AbilityCashCli/config.json` — добавить секции `Timesheet` и пустой `Salaries`.

## Шаги
1. `TimesheetConfig`, `SalaryConfig` (records), интеграция в `AppConfig` + `CreateDefault()`.
2. `TimesheetImporter`: парсинг даты из B2, шапки (по подстрокам), строк данных; ФИО через `PersonNameNormalizer`; извлечение месяца (словарь «январь..декабрь» → 1..12) и года (`\d{4}`) из имени файла; дата операции = последний календарный день месяца + `DefaultTime`.
3. Комментарий транзакции: `"Оклад за {месяц} {год}: {часов} из {нормы} ч"`.
4. `TimesheetWriter`: поиск оклада в `Salaries` (`StringComparer.Ordinal` по ФИО), счёта `[З] ФИО` в БД; сумма = `salary * hours/norm`, округление `AwayFromZero` до 2 знаков; `ExtraComment2 = AbilityCashValues.BuildSourceComment`; `TransactionGroup` с `HolderDateTime = BudgetDate`, `Position` по `MaxAsync`-паттерну как в `VacationsWriter`.
5. `TimesheetRule` по шаблону `VacationsRule`.
6. Регистрация правила в `Program.RunImportAsync` рядом с другими.
7. Правка `config.json`: `Timesheet.SalaryCategoryPath = "Все статьи расхода::Кадры::Заработная плата::Оклад"`, `SalaryAccountPrefix = "[З] "`, `DefaultTime = "12:00"`; `Salaries = []`.

## Риски
- ФИО с одним инициалом («Захарова А.») — `PersonNameNormalizer` оставит как есть (меньше 3 токенов). Ключ в `Salaries` должен совпасть по форме; фатал покажет, какой ключ не найден.
- Месяц в имени только в именительном падеже («март», не «марта»); покрываем 12 вариантов, других форм пока не ожидаем.
- Если норма в табеле отличается между строками — каждая строка считается по своей норме (по факту все равны, но код не завязывается на равенство).
