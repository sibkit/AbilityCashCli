# 015 — BudgetDate всегда 00:00 UTC

## Проблема

Часть импортируемых строк пишет `BudgetDate` с смещением внутри суток (`DefaultTime` из конфигов). У всех существующих записей в базе `BudgetDate` — строго начало суток UTC; AbilityCash падает на смешанном представлении.

## Файлы

- `AbilityCashCli/src/Configuration/CashPayoutsConfig.cs` — убрать `DefaultTime`.
- `AbilityCashCli/src/Configuration/SalaryRegistersConfig.cs` — убрать `DefaultTime`.
- `AbilityCashCli/src/Configuration/TimesheetConfig.cs` — убрать `DefaultTime`.
- `AbilityCashCli/src/Configuration/AppConfig.cs` — убрать строки `DefaultTime = ...` из `CreateDefault`.
- `AbilityCashCli/src/Import/CashPayoutsWriter.cs` — убрать парсинг и поле `_defaultTime`; `budgetDate = StartOfDayUnix(r.Date)`.
- `AbilityCashCli/src/Import/SalaryRegisters/SalaryRegisterWriter.cs` — то же.
- `AbilityCashCli/src/Import/Timesheets/TimesheetImporter.cs` — убрать парсинг и `_defaultTime`; `operationDate = new DateTime(year, month, lastDay)`.
- `AbilityCashCli/src/Import/Timesheets/TimesheetWriter.cs` — `ToUnix` → `StartOfDayUnix`.
- `AbilityCashCli/config.json` — удалить ключи `DefaultTime` из `CashPayouts`, `SalaryRegisters`, `Timesheet`.
- `CLAUDE.md` — зафиксировать инвариант.

## Шаги

1. Удалить поля `DefaultTime` из трёх record-конфигов.
2. Убрать парсинг и использование `_defaultTime` в двух writer'ах и одном importer'е.
3. В местах, где строился `DateTime + _defaultTime`, использовать `AbilityCashValues.StartOfDayUnix(r.Date)` (или просто `r.Date.Date`).
4. Почистить `AppConfig.CreateDefault` и `config.json`.
5. Обновить CLAUDE.md: `Transaction.BudgetDate` — всегда 00:00 UTC, порядок внутри суток — `TransactionGroup.Position`.
