# 022 — Исключить Executed=0 из пересчёта running balance

## Проблема

`AccountBalanceRecalculator` сейчас учитывает все не-удалённые транзакции. AbilityCash в running balance счёта учитывает только `Executed=1` (проведённые); `Executed=0` — план/непроведённая, в running не участвует и не смещает последующие строки. Из-за этого на счётах, где есть `Executed=0` строки, UI-остатки расходятся с нашим пересчётом (наблюдается на `[ЗП] Ледмастер (Альфа)`: 1909 строк со сдвигом +40 825 400 от одной `Executed=0` строки).

## Файлы

- `AbilityCashCli/src/Import/AccountBalanceRecalculator.cs` — добавить фильтр `Executed != 0` в выборку транзакций; `Executed=0` строки не загружаем → их `IncomeBalance`/`ExpenseBalance` не переписываются, `running` не сдвигается.
- `CLAUDE.md` — в секцию про `IncomeBalance`/`ExpenseBalance` добавить, что `Executed=0` в running не участвует.

## Шаги

1. В `RecalculateAsync` к `Where(t => t.Deleted == 0)` добавить `&& t.Executed != 0`.
2. В CLAUDE.md (абзац про `Transactions.IncomeBalance`/`ExpenseBalance`) дописать: `Executed=0` — не участвует в running balance, её `IncomeBalance`/`ExpenseBalance` мы не трогаем.

## Риски

- `Executed=0` строки сохранят старые (возможно некорректные) значения `IncomeBalance`/`ExpenseBalance`. Если пользователю важно обнулять их — отдельный план; текущая задача — только не ломать running для `Executed=1`.
- Все наши писатели создают строки с `Executed=1`, так что свежие импорты не меняются; эффект касается только легаси-данных с `Executed=0`.
