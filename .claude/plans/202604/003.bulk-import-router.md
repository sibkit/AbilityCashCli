# 003 — Пакетный импорт из каталога через роутер

## Проблема

Сейчас CLI умеет один файл через `--import`. Нужен пакетный импорт каталога (`.temp/_extracted/` и др.), с выбором importer по эвристикам (имя/расширение/содержимое), копированием обработанных файлов в `imported/` рядом с exe. Неподдерживаемые файлы — skip+log, не трогаем. Heuristics — легко расширяемы (SOLID: OCP).

## Файлы

- `AbilityCashCli/src/Configuration/AppConfig.cs` — поле `ImportDir` (nullable).
- `AbilityCashCli/src/Configuration/AppConfigJsonContext.cs` — обновить.
- `AbilityCashCli/src/Cli/CliOptions.cs` — `string? ImportDir`.
- `AbilityCashCli/src/Cli/CliArgsParser.cs` — флаг `--import-dir <path>`.
- `AbilityCashCli/src/Import/IImportRule.cs` — `IImporter Importer { get; }` + `bool Matches(string path)`.
- `AbilityCashCli/src/Import/IImportRouter.cs` — `IImporter? Resolve(string path)`.
- `AbilityCashCli/src/Import/ImportRouter.cs` — перебор `IEnumerable<IImportRule>`, first-match-wins.
- `AbilityCashCli/src/Import/Rules/CashPayoutsRule.cs` — regex `^Выплаты наличные.*\.xls$` (ci) → `CashPayoutsImporter`.
- `AbilityCashCli/src/Import/IImportScanner.cs` + `FolderImportScanner.cs` — перечисление файлов каталога (non-recursive), исключая `imported/`.
- `AbilityCashCli/src/Import/IImportArchiver.cs` + `FolderImportArchiver.cs` — move в `{BaseDirectory}/imported/`, на коллизию — суффикс `_yyyyMMdd_HHmmss`.
- `AbilityCashCli/src/Import/BulkImportRunner.cs` — оркестрация: scanner → router → importer → writer → archiver; unsupported/ошибки — log, не архивируем.
- `AbilityCashCli/src/Program.cs` — собрать список rules, router, scanner, archiver, runner. Режимы: `--import <file>` (один файл через ту же трубу) и `--import-dir <path>` / `config.ImportDir` (каталог).

## Шаги

1. `AppConfig.ImportDir` + `AppConfigJsonContext`.
2. `CliOptions.ImportDir` + `--import-dir` в парсере; приоритет CLI > config.
3. Интерфейсы: `IImportRule`, `IImportRouter`, `IImportScanner`, `IImportArchiver`. Реализации — `ImportRouter`, `FolderImportScanner`, `FolderImportArchiver`.
4. `CashPayoutsRule` — инкапсулирует regex + ссылку на свой importer (поле, получает его в ctor).
5. `BulkImportRunner.RunAsync(IEnumerable<string> files)`: для каждого файла — `router.Resolve`; если null → log skip; иначе `importer.Read` → `TransactionWriter.WriteAsync` → `archiver.Archive`. Exception на файле → log, пропускаем, не архивируем, продолжаем.
6. `Program.cs`: собрать `rules = [new CashPayoutsRule(new CashPayoutsImporter())]`, `router = new ImportRouter(rules)`; если `--import <file>` → runner на одном пути; если `--import-dir`/`ImportDir` → `scanner.Enumerate(dir)` → runner.
7. Папку `imported/` при старте не создавать заранее — архивер создаёт по требованию. Scanner пропускает эту папку, если попала внутрь `ImportDir`.

## Риски

- Нет DI-контейнера — rules регистрируем вручную в `Program.cs`. Это локально; OCP держим на уровне списка rules + router.
- `CashPayoutsImporter` сейчас бросает если нет шапки — при ошибке файл не архивируем (runner ловит и пишет log), чтобы можно было починить и перезапустить.
- Коллизия при archive: `file.xls` → `file_20260421_123045.xls`. Нет атомарного перемещения между томами — `File.Move` в рамках одного диска OK; cross-volume пока не поддерживаем.
- `ImportAccountName` — единственный сейчас маппинг счёта, подходит только для cash payouts. Новые importer'ы в будущих планах введут свой маппинг.
- Повторный запуск на том же `ImportDir`: если файл успешно прошёл — уже в `imported/` и не виден scanner'у. Если упал — лежит, ждёт починки.
