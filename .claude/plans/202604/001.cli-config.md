# 001 — CLI-конфиг (config.json)

## Проблема

CLI должен читать настройки из `config.json` рядом с exe (создавая дефолтный, если нет) и позволять перекрывать их CLI-флагами. Пока единственная настройка — путь к Sqlite-БД.

## Файлы

- `AbilityCashCli/src/Program.cs` — переезд из корня; точка входа: загрузка конфига, мердж с CLI, вывод итогового пути к БД.
- `AbilityCashCli/src/Configuration/AppConfig.cs` — record `AppConfig(string DbPath)` с дефолтом.
- `AbilityCashCli/src/Configuration/AppConfigJsonContext.cs` — `JsonSerializerContext` для AOT.
- `AbilityCashCli/src/Configuration/IConfigStore.cs` — интерфейс: `Load()`, путь к файлу.
- `AbilityCashCli/src/Configuration/ConfigStore.cs` — реализация: читает `AppContext.BaseDirectory/config.json`, создаёт дефолтный если нет.
- `AbilityCashCli/src/Cli/CliOptions.cs` — record с nullable-полями (override): `string? DbPath`.
- `AbilityCashCli/src/Cli/CliArgsParser.cs` — парсер `--db <path>` → `CliOptions`.
- `AbilityCashCli/AbilityCashCli.csproj` — удалить старый `Program.cs` из корня (через src layout).

## Шаги

1. Завести `src/` и перенести `Program.cs`.
2. Добавить `AppConfig` (record) + `AppConfigJsonContext` (source-gen JSON).
3. `IConfigStore` + `ConfigStore`: путь = `Path.Combine(AppContext.BaseDirectory, "config.json")`. Если файла нет — сериализовать дефолтный `AppConfig` и записать. Возвращать загруженный.
4. `CliOptions` + `CliArgsParser` (ручной разбор `--db`). Неизвестные флаги — ошибка с подсказкой.
5. В `Program.cs`: `ConfigStore.Load()` → применить `CliOptions` поверх (non-null поля перекрывают) → распечатать итоговый `DbPath`.
6. Подровнять `.csproj` под src layout (если нужно — `<EnableDefaultCompileItems>` оставить, переезд файлов уже учтёт).

## Риски

- AOT + `System.Text.Json`: обязательно через `JsonSerializerContext`, иначе trim-предупреждения.
- Дефолтный `DbPath` — относительный (`./ability.db`)? Уточнить перед шагом 2.
