# AbilityCashCli — CLAUDE.md

## Описание

CLI-инструмент для автоматической обработки БД: импорт из различных форматов файлов, трансформация и сохранение данных.

`использует систему планов`

## Доменная модель AbilityCash

Три типа операций (одна строка `Transactions`):
- **Расход** — `ExpenseAccount` задан, `IncomeAccount` NULL. Статья (`TransactionCategory`) обязательна.
- **Доход** — `IncomeAccount` задан, `ExpenseAccount` NULL. Статья обязательна.
- **Перевод** — оба счёта заданы. Статья опциональна.

Понятие «бюджет» не используем. Поля `BudgetDate` / `BudgetPeriodEnd` в схеме — исторические; трактуем как дату операции и конец её периода.

`TransactionGroup` — поведение не исследовано; уточняется по мере появления задач. AbilityCash капризный к полям — проверять вручную.

**Enterprise** — программный концепт предприятия (`Name` + `PayrollAccount`). В AbilityCash отсутствует; используется на уровне CLI для роутинга источника зарплаты (например, по подстроке имени файла).

## Соглашения импорта

- **Префиксы счетов:** `[ЗП]` — зарплатный счёт предприятия (источник выплат), `[З]` — личный счёт сотрудника (формат `[З] ФИО`).
- **ExtraComment1..4** — только метаданные источника, не пользовательский контент. `ExtraComment2 = "{filename} | {ImporterClassName} v{AppVersion}"` (через `AbilityCashValues.BuildSourceComment`). Версия — `AppInfo.Version`.

## Сборка и тесты

Claude **не запускает** `dotnet build/test/run` — пишет «запусти `dotnet build` / `dotnet test`, кинь итог» и ждёт. Исключение — явное разрешение в текущей переписке; тогда подавлять output: `dotnet build --verbosity quiet --nologo 2>&1 | tail -30`, аналогично для `test`.

## Конвенции

- **Src layout:** исходники `AbilityCashCli/src/`, тесты `AbilityCashCli.Tests/src/`.
- **SOLID:** зависимости через интерфейсы и DI, не прямыми ссылками на реализации.
- **Комментарии:** по умолчанию не пишем. Если нужен — на русском.
- **ProjectReference:** не добавлять новые `<ProjectReference>` без явного разрешения. Сначала предложи, объясни зачем, дождись подтверждения.

## Конфигурация и константы

Литералы-конфигурация (пути, таймауты, форматы) — в Options-record, не в телах методов. Connection strings и креды — только из `IConfiguration`/env, никогда литералом.

Доменные константы (имена/пути категорий, префиксы счетов, коэффициенты расчёта) — через `AppConfig`, не литералом в коде.

## Инструкции (CLAUDE.md)

1. Молчит инструкция — действуй по здравому смыслу. Противоречит задаче — стоп, сначала договариваемся.
2. Claude не правит инструкции сам: предложил → подтвердили → правка → код.
3. Инструкция не коммитится позже соответствующего кода (в один коммит или отдельным раньше).
